# Save Image As Any Type вЂ” v2 Design

## Overview

Expand the extension from a 3-format right-click converter into a full image format toolkit with 8 output formats, batch operations, clipboard support, user-configurable menus, and quality controls. All conversion stays local in the browser. Chrome first, Firefox/Edge later.

## Output Formats

| Format | Method | Library | Notes |
|--------|--------|---------|-------|
| PNG | `canvas.toDataURL("image/png")` | Native | Lossless, preserves alpha |
| JPG | `canvas.toDataURL("image/jpeg", quality)` | Native | Fill background color before draw |
| WebP | `canvas.toDataURL("image/webp", quality)` | Native | Supported in all Chromium browsers |
| AVIF | `canvas.toDataURL("image/avif", quality)` | Native | Chrome 121+; detect support, show error on older |
| TIFF | `UTIF.encodeImage(rgba, w, h)` | UTIF.js | User-selectable compression (None, LZW, Deflate) |
| BMP | Manual byte packing | Inline (~40 lines) | BITMAPFILEHEADER + BITMAPINFOHEADER + raw BGR pixels |
| ICO | Manual byte packing | Inline (~60 lines) | ICONDIR header + embedded PNG payload at target size |
| GIF | gif.js encoder | gif.js | Single frame; quantizes to 256 colors |

### Format-specific details

**ICO:** Embeds a PNG inside the ICO container rather than raw BMP. Supported by all modern systems, keeps code simple. Image gets resized to user's chosen icon size before encoding.

**GIF:** 256-color quantization means visible quality loss on photographs. No warning needed вЂ” users choosing GIF know what they're getting.

**AVIF fallback:** On browsers that don't support AVIF canvas encoding, `toDataURL` returns a PNG. Detect this by checking the returned data URL's MIME type and show an error notification rather than silently saving a PNG.

## Architecture

| File | Purpose |
|------|---------|
| `background.js` | Service worker вЂ” context menu creation, download orchestration, message routing |
| `offscreen.js` | Image conversion engine вЂ” all format encoding happens here |
| `offscreen.html` | DOM host for offscreen canvas operations |
| `options.html` + `options.js` | Settings UI вЂ” format visibility, quality defaults, notification prefs |
| `content.js` | Content script вЂ” page image scanning for batch ops, modifier key capture |
| `lib/UTIF.js` | TIFF encoding (existing) |
| `lib/gif.js` | GIF encoding (new) |

No bundler, no build step. BMP and ICO encoding are inline in `offscreen.js`.

### New permissions required

The following must be added to `manifest.json`:

| Permission | Reason |
|-----------|--------|
| `storage` | `chrome.storage.sync` for user settings |
| `notifications` | Chrome notifications for error/batch feedback |
| `clipboardWrite` | "Copy Image As..." clipboard feature |
| `scripting` | Inject content script for batch page scanning and modifier key capture |

### Shift+click shortcut

Holding Shift while clicking any "Save" format copies to clipboard instead of saving to disk. This is documented as a tip in the options page.

**Implementation note:** Chrome's `contextMenus.OnClickData` does not include modifier key state (unlike Firefox which has a `modifiers` property). The workaround is: `content.js` listens for the `contextmenu` DOM event (fires before the browser menu appears) and captures `event.shiftKey`. This modifier state is sent to the service worker via `chrome.runtime.sendMessage`, and when `onClicked` fires, the most recently captured modifier state is used to decide between save and clipboard.

### Context types

All four top-level items are separate menu entries (not nested under a shared parent):
- "Save Image As..." вЂ” `contexts: ["image"]`
- "Copy Image As..." вЂ” `contexts: ["image"]`
- "Save All Images As..." вЂ” `contexts: ["image", "page"]` (visible on both image and page right-clicks)
- "Customize..." вЂ” `contexts: ["image", "page"]`

## Options Page

Three sections:

### 1. Format Settings

Each format gets a row with:
- Drag handle for reorder
- Format name
- Visibility toggle (show/hide from menu)
- Placement: "Top level" or "Others submenu"
- Expandable format-specific settings:
  - **JPG:** Quality slider (1-100, default 92)
  - **WebP:** Quality slider (1-100, default 90)
  - **AVIF:** Quality slider (1-100, default 80)
  - **PNG:** No settings (always lossless)
  - **TIFF:** Compression dropdown (None, LZW, Deflate вЂ” default Deflate)
  - **BMP:** No settings
  - **ICO:** Size dropdown (16, 32, 48, 64, 128, 256 вЂ” default 256)
  - **GIF:** No settings (single frame)

### 2. Transparency

- Background color picker for formats that don't support alpha (JPG, BMP)
- Default: white
- Presets: white, black, with transparent checkerboard preview

### 3. Notifications

- Toggle: Show success badge (default: on)
- Toggle: Show error notifications (default: on)
- Default action: "Save to disk" / "Copy to clipboard" dropdown
- Tip: "Hold Shift while clicking any format to quickly switch between save and copy."

All settings sync across devices via `chrome.storage.sync`.

## Batch Operations

### Trigger

"Save All Images As..." submenu in context menu. Appears on right-click anywhere on the page.

### Flow

1. User right-clicks page в†’ "Save All Images As..." в†’ picks format
2. `background.js` injects a content script that scans the page for:
   - `<img>` elements with `src` attribute
   - `<picture>` source elements
   - CSS `background-image` URLs
3. Deduplicates URLs, filters out tiny images (<10px, tracking pixels)
4. Shows Chrome notification: "Found 23 images. Saving as PNG..."
5. Converts and downloads sequentially (avoids memory spikes)
6. Each file uses original filename; collisions handled via `conflictAction: "uniquify"` (appends (1), (2), etc.)
7. On completion: badge + notification showing "Saved 23/23 images" or "Saved 21/23 (2 failed)"

### Guardrails

- If more than 50 images found, notification asks for confirmation first
- If a single image fails, log the error and continue with the rest
- User can close the notification to cancel remaining downloads

### Permissions

Scanning for CSS background images requires injecting a content script via `chrome.scripting.executeScript()`, authorized by the existing `<all_urls>` host permission and the new `scripting` permission.

## Error Handling & Feedback

### Success

- Badge: green checkmark on extension icon for 2 seconds, then clears
- Clipboard success: green badge + brief notification "Copied to clipboard as PNG"
- No notification for saves (unless user wants it)

### Errors

- Badge: red X for 3 seconds
- Chrome notification with actionable message:
  - "Failed to save image: could not fetch (CORS blocked)"
  - "Failed to save image: format not supported in this browser (AVIF)"
  - "Failed to save image: file too large for canvas"
- Auto-dismisses after 5 seconds

### Batch

- Badge shows running count: "3/23"
- On completion: green "23" or yellow "21/23" for partial failures
- Notification summarizes results; lists failed filenames on click

### User control

All notifications respect toggles from options page. If user turns off success badges, only errors show. If user turns off everything, extension is completely silent.

## Browser Port Strategy (Future)

Not part of this release.

**Chrome в†’ Edge:** Zero changes. Same Manifest V3, same APIs. Submit to Edge Add-ons store.

**Chrome в†’ Firefox:**
- Replace `chrome.offscreen` (doesn't exist in Firefox) with hidden iframe or direct canvas access
- Add `browser_specific_settings.gecko.id` to manifest
- `chrome.storage.sync` works but has different quota limits
- Context menu API is identical

Estimated effort: a few hours of adaptation, not a rewrite.
